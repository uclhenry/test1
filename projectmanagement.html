Projects features

we will enable the user to select the disease which they wish to analyse. Based on the disease selected, the user will then be asked to select which widgets they wish to view. Some widgets or data sources will be disease-specific, such as Influenzanet and Flu survey in the example, while others such as twitter will apply to all diseases.
The widgets will then be displayed in the panel to the right. A heat map will also be displayed, showing the geographical location of each entry from the data source. A layer selector is also available, for users to choose which data sources they wish to view on the heat map, allowing them to combine different data sources as well.
The data will also be passed automatically from the database to the risk assessment algorithms below. However, we will still allow user input to supplement the data received from various sources. The view report button will allow users to see a draft of the daily report which will be generated based on the available data at that point in time.

Requirements Specification


Data Display & visualisation

1 The system should enable use of different data sources for different diseases M

2 The system should enable addition of other widgets M

3 The system should be able to present data with large difference in values on the same chart M

4 The heat map should visualise each data source in a different layer and users should be able to select layers M

5 The system should differentiate between possible cases, probable cases and confirmed cases M

6 The system should offer predictions on the future possible spread of diseases or vectors C


Risk Assessment

7 The system should pass data into the risk assessment algorithm automatically M

8 The system should automatically generate reports for diseases daily for experts S

9 The system should implement some simple filtering to remove false positives from data M

10 The system should have discussion forums set up S
